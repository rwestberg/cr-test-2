 1 # Contributing
 2 
 3 Thank you for considering contributing to the
 4 [JDK Mission Control](https://openjdk.java.net/projects/jmc) project! For information 
 5 about contributing to [OpenJDK](https://openjdk.java.net/) projects, which include
 6 JDK Mission Control, please see <https://openjdk.java.net/contribute/>.
 7 
 8 ## Pull Requests
 9 
10 The JDK Mission Control project (JMC) gladly accepts contributions in the form of pull 
11 requests on [GitHub](https://github.com/openjdk/jmc/pulls/).



12 
13 ## Mailing List
14 
15 The mailing list for JDK Mission Control is `jmc-dev@openjdk.java.net`. See
16 <https://mail.openjdk.java.net/mailman/listinfo/jmc-dev> for instructions
17 on how to subscribe of if you want to read the archives.
18 
19 ## Issues
20 
21 You can find open issues to work on in the JMC project in the
22 [JDK Bug System](https://bugs.openjdk.java.net/):
23 <https://bugs.openjdk.java.net/secure/Dashboard.jspa?selectPageId=13519>.
24 
25 ## Larger Contributions
26 
27 If you have a larger contribution in mind then we highly encourage you to first
28 discuss your changes on the JMC mailing list, `jmc-dev@openjdk.java.net`,
29 _before_ you start to write the code.

 8 ## Pull Requests
 9 
10 The JDK Mission Control project (JMC) gladly accepts contributions in the form of pull 
11 requests on [GitHub](https://github.com/openjdk/jmc/pulls/). Ensure that the commit 
12 message refers to an existing JMC issue in the [JDK Bug System](https://bugs.openjdk.java.net/).
13 The format is "&lt;bug id number&gt;: Commit message". For example, for a PR for issue JMC-6789, the commit 
14 message should be "6789: Commit message".
