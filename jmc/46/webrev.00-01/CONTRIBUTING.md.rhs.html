1 # Contributing
 2 
 3 Thank you for considering contributing to the
 4 [JDK Mission Control](https://openjdk.java.net/projects/jmc) project! For information 
 5 about contributing to [OpenJDK](https://openjdk.java.net/) projects, which include
 6 JDK Mission Control, please see <https://openjdk.java.net/contribute/>.
 7 
 8 ## Pull Requests
 9 
10 The JDK Mission Control project (JMC) gladly accepts contributions in the form of pull 
11 requests on [GitHub](https://github.com/openjdk/jmc/pulls/). Ensure that the commit 
12 message refers to an existing JMC issue in the [JDK Bug System](https://bugs.openjdk.java.net/).
13 The format is "&lt;bug id number&gt;: Commit message". For example, for a PR for issue JMC-6789, the commit 
14 message should be "6789: Commit message".  
15 
16 ## Mailing List
17 
18 The mailing list for JDK Mission Control is `jmc-dev@openjdk.java.net`. See
19 <https://mail.openjdk.java.net/mailman/listinfo/jmc-dev> for instructions
20 on how to subscribe of if you want to read the archives.
21 
22 ## Issues
23 
24 You can find open issues to work on in the JMC project in the
25 [JDK Bug System](https://bugs.openjdk.java.net/):
26 <https://bugs.openjdk.java.net/secure/Dashboard.jspa?selectPageId=13519>.
27 
28 ## Larger Contributions
29 
30 If you have a larger contribution in mind then we highly encourage you to first
31 discuss your changes on the JMC mailing list, `jmc-dev@openjdk.java.net`,
32 _before_ you start to write the code.
33 
34 ## Slack
35 
36 The slack channel for JMC developers can be found here:
37 <https://jdkmissioncontrol.slack.com/>
38 
39 If you're working for a company already contributing, you can sign up here:
40 <https://jdkmissioncontrol.slack.com/signup>
41 
42 If not, ask for an invite in the jmc-dev mailing list.
43 
44 ## Questions
45 
46 If you have a question or need help, please send an email to our mailing list
47 `jmc-dev@openjdk.java.net` or stop by the JMC slack channel.
